# Software-Security
For this artifact, the client was Artemis Financial, and the goal was to improve the security of a Spring Boot REST service by identifying and addressing dependency vulnerabilities. I configured the OWASP Dependency-Check Maven plugin within the project’s pom.xml and generated an initial vulnerability report. The scan identified a CVE associated with the jackson-databind dependency. After reviewing the report, I analyzed whether the vulnerability was applicable to the project’s actual implementation. Since the CVE involved an XML entity expansion scenario and this application strictly processes JSON without enabling XML parsing features, the vulnerability did not apply to the way the software was configured.

To address this properly, I created a suppression.xml file and added a rule to suppress CVE-2020-25649 for the jackson-databind dependency. I then updated the dependency-check plugin configuration to reference the suppression file and reran the Maven build to generate an updated report. The new report confirmed “Vulnerabilities Suppressed: 1,” demonstrating that the false positive was handled correctly. This process strengthened my understanding of how to evaluate vulnerability findings instead of blindly applying updates, and it reinforced the importance of understanding context when assessing security risks.

After making the configuration changes, I rebuilt the project to ensure it remained functional and reran the dependency scan to verify that no additional vulnerabilities were introduced. This assignment demonstrates my ability to use industry-standard tools such as OWASP Dependency-Check, analyze CVEs critically, configure Maven plugins securely, and document mitigation decisions clearly. For future employers, this artifact shows that I can assess software security risks, justify suppression decisions responsibly, and maintain both security and functionality within a production-style application.
